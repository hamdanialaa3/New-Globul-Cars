# خطة الإصلاح الشاملة لمنصة New Globul Cars

هذا المستند يحدد خطة تنفيذية عملية لتحسين الاعتمادية والأداء والاتساق المعماري في الواجهة (React)، الوظائف الخلفية (Firebase Functions)، وخدمة الذكاء الاصطناعي.

## 1. التكاملات الخارجية

التكاملات: خرائط جوجل، hCaptcha، Supabase، Facebook Pixel، OAuth، Socket.io.

### الأهداف

- ضمان تهيئة صحيحة وفشل واضح مع Fallback.
- منع الأعطال الصامتة (Silent Failures).

### الإجراءات

1. سكربت Startup Health يتحقق من كل مفتاح بيئة ويعيد JSON حالة.
2. تفعيل Leaflet تلقائياً عند غياب مفتاح خرائط جوجل.
3. إضافة طبقة إعادة المحاولة مع backoff لـ Socket.io.
4. إحاطة Pixel/OAuth بشرط موافقة الخصوصية (إن وُجدت سياسة مستقبلية).
5. تقييم استخدام Supabase (إن كان <5% من مسارات الملفات يمكن تعطيله افتراضياً).

### مؤشرات نجاح (KPIs)

- 100% نجاح تهيئة في التطوير والإنتاج.
- زمن فحص التهيئة < 300ms.

## 2. إدارة المفاتيح والسرية

### الإجراءات

1. إنشاء `scripts/audit-env.ts` لفحص `.env` مقابل قائمة مطلوبة.
2. منع طباعة أي متغير يبدأ بـ `REACT_APP_FIREBASE_*` في الإنتاج.
3. خطوة CI تفشل عند وجود مفتاح ناقص.

### KPIs

- صفر مفاتيح ناقصة في البناء.
- صفر تسريبات في السجلات.

## 3. نمط Singleton للخدمات

### الإجراءات

1. مراجعة كل خدمة والتأكد من وجود `private constructor` أو حراسة داخل `getInstance()`.
2. إضافة اختبار واحد يقيس عدد المثيلات بعد استدعاءات متعددة.
3. قاعدة تحليل بسيطة تمنع `new SomeService()` خارج ملفه.

### KPIs

- 0 حالات إنشاء غير مسيطر عليها.

## 4. تقسيم الكود والأداء

### الإجراءات

1. استخراج تقرير Webpack (stats) قبل/بعد.
2. إضافة Lazy boundaries حول صفحات: AnalyticsDashboard, BillingPage, TeamManagement إن لم تكن بالفعل مثالية.
3. Prefetch ثاني خطوة في تدفق البيع اعتماداً على نوع المركبة المختارة.
4. مراقبة FCP و LCP بعد كل دفعة.

### KPIs

- تقليل حجم الحزمة المبدئية ≥ 20%.
- FCP < 2s على جهاز متوسط.

## 5. نظام التصميم (Theme + Font)
### الإجراءات
1. فحص جميع ملفات CSS/Styled للتأكد من استخدام خط 'Martica'.
2. استخراج الألوان الصلبة المتكررة إلى `theme.ts` (إن وجدت خارجاً).
3. منع أنماط معقدة Inline في الصفحات الثقيلة (تحذير في مراجعة الكود / لاحقاً Rule).
### KPIs
- كل الألوان الحرجة ضمن theme.
- 0 خطوط متباينة.

## 6. التوحيد في التسجيل Logging
### الإجراءات
1. بحث عن `console.` واستبدال بـ `logger.*`.
2. إضافة فلاتر تمنع تمرير كائنات حساسة.

## 15. أدوات التنفيذ (Scripts)
سيتم إنشاء سكربتات في `bulgarian-car-marketplace/scripts/`:
- `audit-env.ts`: فحص المفاتيح.
- `check-translations.ts`: تدقيق الترجمة.
- `check-provider-order.ts`: اختبار ترتيب المزودين.
- `scan-realtime-cleanup.ts`: تحليل الاشتراكات.
- `scan-legacy-location-fields.ts`: تحذير استخدام الحقول القديمة.
