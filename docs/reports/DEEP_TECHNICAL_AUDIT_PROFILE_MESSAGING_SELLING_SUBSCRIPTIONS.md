### المشكلة 3: نظام الـ Numeric ID نفسه قد يكون معيب

#### الأخطاء المحتملة:

**3.1 - Race Condition في تعيين الـ Numeric ID:**
```
❌ المشكلة:
- ensureUserNumericId() تستخدم transaction لكن:
  - إذا فشل التحديث الأول: retries 5 مرات
  - قد يحصل المستخدم على numeric ID مختلف في المحاولة الثانية
  - أو قد لا يحصل على أي numeric ID

✅ الحل المطلوب:
- إضافة unique constraint على حقل numericId
- التأكد من أن كل user له numeric ID واحد فقط
- إضافة logging شامل لتتبع كل عملية تعيين
```

**3.2 - عدم وجود آلية للتحقق من الـ Numeric ID:**
```
❌ المشكلة:
- لا يوجد endpoint للتحقق من أن numeric ID موجود بالفعل
- قد يحاول المستخدم الوصول إلى /profile/999999 (موجود؟)
- لا يوجد معالجة للـ 404

✅ الحل المطلوب:
- إضافة numeric ID validation service
- إرجاع 404 إذا كان numeric ID غير موجود
- تصريح واضح في الـ UI عند محاولة الوصول لـ numeric ID غير موجود
```

**3.3 - المستخدمون القدماء قد لا يملكون Numeric ID:**
```
❌ المشكلة:
- ensureUserNumericId() تُستدعى فقط في ProfilePageWrapper
- إذا كان لديك users قدماء بدون numeric ID
- قد يحصل على واحد في المرة التالية التي يزور فيها ملفه
- لكن هذا قد يتغير في كل مرة؟ (race condition)

✅ الحل المطلوب:
- إنشاء migration script يعين numeric ID لكل المستخدمين القدماء
- التأكد من أن كل user له numeric ID واحد ثابت
- اختبار شامل قبل التطبيق
```
