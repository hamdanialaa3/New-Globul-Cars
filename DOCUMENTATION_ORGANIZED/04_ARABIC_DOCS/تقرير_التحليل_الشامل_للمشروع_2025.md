### 1. مشاكل الأمان (Security Issues)

#### 1.1 Firestore Rules - قراءة عامة للبيانات الحساسة
**الموقع:** `firestore.rules`
**المشكلة:**
```javascript
match /users/{userId} {
  allow read: if isAuthenticated(); // ⚠️ أي مستخدم يقرأ كل البيانات!
}
```
**الخطر:** أي مستخدم مسجل يمكنه قراءة بيانات جميع المستخدمين (إيميلات، أرقام هواتف، مواقع)

**الحل المقترح:**
```javascript
match /users/{userId} {
  // قراءة عامة للحقول العامة فقط
  allow read: if isAuthenticated() && 
    (request.auth.uid == userId || 
     resource.data.profileVisibility == 'public');
  
  // قراءة محدودة للبيانات العامة
  function getPublicFields() {
    return {
      displayName: resource.data.displayName,
      photoURL: resource.data.photoURL,
      profileType: resource.data.profileType,
      stats: resource.data.stats,
      // فقط الحقول العامة
    };
  }
}
```

#### 1.2 عدم وجود Rate Limiting
**الموقع:** متعدد (مثل `followService`, `messageService`)
**المشكلة:**
```typescript
const handleFollow = async (userId: string) => {
  await followService.followUser(currentUser.uid, userId);
  // ⚠️ يمكن الضغط 100 مرة في الثانية!
};
```
**الخطر:** Spam attacks, استنزاف Firestore writes, إرهاق النظام

**الحل المقترح:**
```typescript
import { useThrottle } from '@/hooks/useThrottle';

const throttledFollow = useThrottle(handleFollow, 1000); // Max once per second

// أو في Backend:
// functions/src/rate-limiting/rateLimiter.ts
export const rateLimiter = functions.https.onCall(async (data, context) => {
  const userId = context.auth?.uid;
  const action = data.action;
  
  const key = `rate_limit:${userId}:${action}`;
  const count = await redis.incr(key);
  if (count === 1) await redis.expire(key, 60); // 60 seconds window
  
  if (count > 10) { // Max 10 actions per minute
    throw new functions.https.HttpsError('resource-exhausted', 'Rate limit exceeded');
  }
});
```
