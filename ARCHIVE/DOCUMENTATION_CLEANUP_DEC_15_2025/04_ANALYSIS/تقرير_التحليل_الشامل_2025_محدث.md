### 3. مشاكل React Hooks

#### 3.1 useEffect بدون dependencies صحيحة

**الحالة:** 🟡 متوسطة

**المشكلة:**
```typescript
// خطأ شائع:
useEffect(() => {
  fetchData();
  // ❌ dependencies مفقودة
}, []);

// أو:
useEffect(() => {
  applyFilters();
}, [users, searchTerm]); // ⚠️ قد تكون dependencies غير كاملة
```

**الحل المقترح:**
```typescript
useEffect(() => {
  fetchData();
}, [/* dependencies صحيحة */]);

// أو استخدام useCallback:
const fetchData = useCallback(async () => {
  // logic
}, [dependencies]);

useEffect(() => {
  fetchData();
}, [fetchData]);
```

#### 3.2 Memory Leaks المحتملة

**الحالة:** 🟡 متوسطة

**المشكلة:**
- بعض `useEffect` لا تقوم بتنظيف الـ listeners
- Firestore listeners قد لا يتم إلغاء الاشتراك فيها بشكل صحيح

**مثال:**
```typescript
useEffect(() => {
  const unsubscribe = onSnapshot(query, (snapshot) => {
    // logic
  });
  // ❌ قد يكون cleanup مفقود في بعض الحالات
  return () => unsubscribe();
}, []);
```

### 4. مشاكل Firebase/Firestore

#### 4.1 Null Safety في Firestore Queries

**الحالة:** ✅ تم إصلاح معظمها

**ملاحظة:** تم إصلاح معظم المشاكل المتعلقة بـ null safety في Firestore listeners، لكن قد توجد حالات أخرى.

#### 4.2 Firestore Rules غير كاملة

**الحالة:** 🟡 متوسطة

**المشاكل:**
```javascript
// في firestore.rules:
allow read: if true; // ⚠️ قد يكون مفتوحاً أكثر من اللازم

// بعض المجموعات لا تتحقق من isActive
match /cars/{carId} {
  allow read: if true; // ❌ يجب التحقق من status == 'active'
}
```

**الحل المقترح:**
```javascript
match /cars/{carId} {
  allow read: if resource.data.status == 'active' 
              && resource.data.isActive == true
              && !resource.data.isSold;
}
```

import { z } from 'zod';

const CarListingSchema = z.object({
  make: z.string().min(1),
  model: z.string().min(1),
  price: z.number().positive(),
  year: z.number().int().min(1900).max(2100),
});

type CarListing = z.infer<typeof CarListingSchema>;

// في الـ service:
function createCarListing(data: unknown): CarListing {
  return CarListingSchema.parse(data); // throws if invalid
}
