## ⚠️ المشكلة 2: CORS Error - يحتاج إجراء منك

### المشكلة:
```
Access to image at 'https://firebasestorage.googleapis.com/...' 
has been blocked by CORS policy: 
No 'Access-Control-Allow-Origin' header is present
```

### السبب:
- Firebase Storage لا يسمح بتحميل الصور من localhost
- CORS غير مهيأ لـbucket التخزين

### الحلول المتاحة (اختر واحد):

#### **الحل 1: عبر Firebase Console (الأسهل)** ⭐ يُنصح به

**الخطوات:**

1. اذهب إلى [Firebase Console](https://console.firebase.google.com/)
2. اختر مشروعك: **fire-new-globul**
3. من القائمة اليسرى → **Storage**
4. اضغط على تبويب **Rules**
5. استبدل القواعد بهذه:

```
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    // السماح بالقراءة للجميع
    match /{allPaths=**} {
      allow read: if true;
      // الكتابة للمستخدمين المسجلين فقط
      allow write: if request.auth != null;
    }
  }
}
```

6. اضغط **Publish**
7. انتظر دقيقة
8. أعد تحميل الصفحة

#### **الحل 2: عبر gsutil (متقدم)**

إذا كان لديك Google Cloud SDK:

```bash
# 1. تسجيل دخول
gcloud auth login

# 2. تعيين المشروع
gcloud config set project fire-new-globul

# 3. تطبيق إعدادات CORS
gsutil cors set cors.json gs://fire-new-globul.firebasestorage.app
```

**ملاحظة:** ملف `cors.json` تم إنشاؤه بالفعل في مجلد المشروع

#### **الحل 3: عبر Firebase CLI (بديل)**

```bash
# 1. نشر قواعد Storage المحدثة
firebase deploy --only storage:rules

# 2. انتظر دقيقة
# 3. أعد تحميل الصفحة
```

**الملفات المنشأة:**
- ✅ `cors.json` - إعدادات CORS
- ✅ `FIX_CORS_FIREBASE.md` - دليل مفصل
- ✅ `public/.htaccess` - إعدادات CORS للـserver
