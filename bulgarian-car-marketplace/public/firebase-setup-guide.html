🔥 دليل إصلاح Firebase Authentication

خطوات إصلاح مشاكل تسجيل الدخول

🚨 المشاكل الحالية

  • ❌ خطأ داخلي في Firebase (auth/internal-error)
  • ❌ فشل Google Sign-in
  • ❌ فشل Facebook Sign-in
  • ❌ فشل Apple Sign-in
  • ❌ مشكلة Firebase App Check Token

🔥 1. إصلاح Google Sign-in (أولوية قصوى)

السبب الأكثر احتمالاً: Google Sign-in Provider غير مفعل في Firebase Console

🔧 فتح Firebase Console - Authentication

الخطوات:

  • اذهب للرابط أعلاه
  • ابحث عن "Google" في قائمة Sign-in providers
  • تأكد من أن الحالة "Enabled" (مفعّل)
  • إذا كان معطل، اضغط على القلم للتحرير
  • اضغط "Enable" ثم "Save"

📍 2. إضافة Authorized Domains

🌐 فتح Authentication Settings

النطاقات المطلوبة:

localhost 127.0.0.1 studio-448742006-a3493.firebaseapp.com

الخطوات:

  • في قسم "Authorized domains"
  • تأكد من وجود النطاقات أعلاه
  • إذا لم تكن موجودة، اضغط "Add domain"

🔑 3. فحص Google Cloud Console

☁️ فتح Google Cloud Console

التحقق من OAuth 2.0 Client IDs:

  • يجب وجود Web application client
  • Authorized redirect URIs يجب أن تحتوي على:
https://studio-448742006-a3493.firebaseapp.com/__/auth/handler http://localhost:3000

✅ 4. Firebase App Check - تم الإصلاح

تم تعطيل Firebase App Check بالكامل لحل مشكلة auth/firebase-app-check-token-is-invalid

ملاحظة: يمكن إعادة تفعيل App Check لاحقاً مع إعدادات reCAPTCHA صحيحة إذا لزم الأمر.

🧪 5. اختبار النظام

🚀 صفحة اختبار المصادقة

استخدم هذه الصفحة لاختبار جميع طرق تسجيل الدخول مع تسجيل مفصل للأخطاء.

🔧 6. التشخيص المتقدم

🔍 صفحة التشخيص الشاملة

إذا استمرت المشاكل، استخدم صفحة التشخيص للحصول على تحليل تفصيلي.

⚡ خطة العمل السريعة (5 دقائق)

  1. الخطوة 1: افتح Firebase Console وفعّل Google Sign-in Provider
  2. الخطوة 2: تحقق من Authorized Domains
  3. الخطوة 3: اختبر النظام في صفحة الاختبار
  4. الخطوة 4: إذا لم تحل المشكلة، فحص Google Cloud Console

📞 إذا استمرت المشاكل

إذا جربت كل الخطوات أعلاه ولم تحل المشكلة:

  • 🌐 جرب المشروع من جهاز/شبكة مختلفة
  • 🧹 امسح cache المتصفح وأعد تحميل الصفحة
