2. المكوّنات المعمارية الأساسية
مكوّنات النظام

Event Trigger: عند اكتمال نشر الإعلان في النظام (post-publish hook).

Publisher Service (خدمة مستقلة أو Cloud Function): تستقبل الحدث، تجمع البيانات، وتدفعه إلى صف انتظار.

Processing Queue (مثلاً RabbitMQ / Cloud Tasks / PubSub): يضمن معالجة متوازية ومقاومة للأخطاء.

Content Generator: وحدة توليد المحتوى الذكي (Local Model + rules engine) تنتج: عنوان، وصف مختصر، CTA، هاشتاغات، اقتراح صورة رئيسية، نصوص بديلة للصور، اقتراحات جدولة.

Media Processor: يضغط الصور، يولّد صور مصغرة، ويصنع فيديو slideshow إن طُلب.

Social Publisher: مكوّن يتكلم مع Meta Graph API لنشر المنشور (Facebook Page / Instagram Feed / IG Carousel / Reels إذا مطلوب).

Tracker & Analytics: يضيف UTM إلى روابط، يسجل معرف المنشور في DB، يجمع إحصاءات النقرات والتفاعل.

Admin Dashboard: واجهة لمراجعة الاقتراحات، الموافقة اليدوية، وإعادة النشر أو التراجع.

Logs & Monitoring: Sentry, Cloud Logging, Alerts.

تدفق البيانات باختصار

المستخدم ينشر إعلان → حدث ad.published يُرسل.

Publisher Service يضع مهمة في Queue مع بيانات الإعلان.

Worker يلتقط المهمة → Media Processor يجهز الصور/فيديو → Content Generator يولّد نصوص متعددة.

Social Publisher ينشر إلى Facebook/Instagram عبر Graph API، يسجّل معرف المنشور.

Tracker يضيف UTM ويبدأ جمع المقاييس.

إذا فشل النشر، يُعاد المحاولة وفق سياسة backoff أو يُرسل تنبيه للمشرف.

js
// pseudo-code: call local LLM service
import fetch from 'node-fetch';

async function generateAdText(ad) {
  const prompt = `
  Generate two variants for a social post in Arabic for a car ad.
  Include title (short), description (1-2 sentences), 3 hashtags, and a CTA.
  Ad data: ${JSON.stringify(ad)}
  `;
  const res = await fetch('http://localhost:5000/generate', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ prompt, max_tokens: 200 })
  });
  const json = await res.json();
  return json.text; // parse into structured fields
}

// pseudo-code for publishing an image post to Facebook Page
import fetch from 'node-fetch';

async function publishToFacebook(pageAccessToken, pageId, message, imageUrl, link) {
  // 1) Upload photo
  const uploadRes = await fetch(`https://graph.facebook.com/${pageId}/photos`, {
    method: 'POST',
    body: new URLSearchParams({
      url: imageUrl,
      caption: message,
      access_token: pageAccessToken
    })
  });
  const uploadJson = await uploadRes.json();
  // 2) Optionally attach link in post or in caption
  return uploadJson; // contains post_id or id
}

2. المكونات الأساسية التي أنفّذها وماذا تفعل كل واحدة
Publisher Service

يستقبل حدث ad.published من تطبيقك (webhook أو internal event).

يجمع بيانات الإعلان الأساسية ويضع مهمة في صف انتظار (Redis Queue / BullMQ).

يكتب سجل أولي في DB مع حالة queued.

Worker

يلتقط المهمة من الصف ويشغّل سلسلة خطوات متسلسلة:

Media Processor: يضغط الصور، يولّد signed URLs، يصنع فيديو slideshow إن طُلب.

Content Generator Client: يستدعي النموذج المحلي عبر واجهة HTTP للحصول على JSON منسق: { title, description, hashtags, cta, variants }.

Social Publisher: ينشر إلى Facebook Page وInstagram Business عبر Meta Graph API، يسجّل post_id وstatus.

يدير retries مع exponential backoff ويحدّد أخطاء قابلة للتصحيح أو فشل دائم.

LLM API Wrapper (Local Model)

واجهة بسيطة (Flask) تستدعي نموذج محلي مثل Llama2/Mistral عبر text-generation-webui أو llama.cpp..

تقبل JSON input وتُرجع JSON output منسق وفق schema محدد.

تحتوي على prompts جاهزة ومُهيأة بالعربية والإنجليزية.

import requests

    payload = request.json

    # call local LLM backend
    resp = requests.post(f"{LLM_BACKEND}/api/generate", json={"prompt": prompt, "max_new_tokens": 200})
    text = resp.json().get('text', '')
    # parse text into JSON according to schema or expect model to return JSON

3.6 Social Publisher snippet services/worker/src/steps/socialPublisher.ts
ts
import fetch from 'node-fetch';

async function publishToFacebook(pageId: string, token: string, message: string, imageUrl?: string) {
  // upload photo
  const photoRes = await fetch(`https://graph.facebook.com/${pageId}/photos`, {
    method: 'POST',
    body: new URLSearchParams({
      url: imageUrl,
      caption: message,
      access_token: token
    })
  });
  const photoJson = await photoRes.json();
  if (photoJson.error) throw new Error(JSON.stringify(photoJson.error));
  return photoJson; // contains id
}

async function publishToInstagram(igUserId: string, token: string, caption: string, imageUrl: string) {
  // create media object
  const createRes = await fetch(`https://graph.facebook.com/${igUserId}/media`, {
    method: 'POST',
    body: new URLSearchParams({
      image_url: imageUrl,
      caption,
      access_token: token
    })
  });
  const createJson = await createRes.json();
  if (createJson.error) throw new Error(JSON.stringify(createJson.error));
  // publish container
  const publishRes = await fetch(`https://graph.facebook.com/${igUserId}/media_publish`, {
    method: 'POST',
    body: new URLSearchParams({
      creation_id: createJson.id,
      access_token: token
    })
  });
  return publishRes.json();
}

إن كانت هناك صورة داخلية وخارجية، فضّل صورة خارجية تُظهر السيارة كاملة.

ضغط الصور: استخدم sharp في Node.js  لضغط وتوليد thumbnails.

إنشاء فيديو slideshow: استخدم ffmpeg لدمج الصور مع نصوص قصيرة من الـ LLM كـ captions.

مثال mediaProcessor.ts:

ts
import sharp from 'sharp';
import { execSync } from 'child_process';

export async function processImages(imagePaths: string[]) {
  const processed = [];
  for (const p of imagePaths) {
    const out = p.replace('.jpg', '.webp');
    await sharp(p).resize(1200).webp({ quality: 80 }).toFile(out);
    processed.push(out);
  }
  return processed;
}

export function createSlideshow(images: string[], outFile: string) {
  const inputs = images.map(i => `-loop 1 -t 2 -i ${i}`).join(' ');
  const filter = `-filter_complex "concat=n=${images.length}:v=1:a=0,format=yuv420p"`;
  execSync(`ffmpeg ${inputs} ${filter} -y ${outFile}`);
}

Publisher Service services/publisher_service/src/index.ts
ts
import express from 'express';
import bodyParser from 'body-parser';
import { Queue } from 'bullmq';
import IORedis from 'ioredis';
import dotenv from 'dotenv';
dotenv.config();

const app = express();
app.use(bodyParser.json());

const redis = new IORedis(process.env.REDIS_URL);
const queue = new Queue('ad_publish_queue', { connection: redis });

app.post('/webhook/ad_published', async (req, res) => {
  const ad = req.body;
  if (!ad || !ad.id) return res.status(400).send('invalid payload');
  await queue.add('publish', { ad });
  return res.status(202).json({ queued: true, adId: ad.id });
});

import sharp from 'sharp';
import fs from 'fs';
import fetch from 'node-fetch';
import path from 'path';
import { execSync } from 'child_process';
import dotenv from 'dotenv';
dotenv.config();

const TMP = process.env.MEDIA_TEMP_DIR || './tmp_media';
if (!fs.existsSync(TMP)) fs.mkdirSync(TMP, { recursive: true });

export async function downloadImage(url: string, outPath: string) {
  const res = await fetch(url);
  const buffer = await res.buffer();
  fs.writeFileSync(outPath, buffer);
}

export async function processImages(imageUrls: string[]) {
  const processed: string[] = [];
  for (let i = 0; i < imageUrls.length; i++) {
    const url = imageUrls[i];
    const tmpFile = path.join(TMP, `img_${Date.now()}_${i}.jpg`);
    await downloadImage(url, tmpFile);
    const out = tmpFile.replace('.jpg', '.webp');
    await sharp(tmpFile).resize(1200).webp({ quality: 80 }).toFile(out);
    processed.push(out);
  }
  return processed;
}

export function createSlideshow(images: string[], outFile: string) {
  // simple ffmpeg slideshow, requires ffmpeg installed
  const inputs = images.map(i => `-loop 1 -t 2 -i "${i}"`).join(' ');
  const filter = `-filter_complex "concat=n=${images.length}:v=1:a=0,format=yuv420p"`;
  execSync(`ffmpeg ${inputs} ${filter} -y "${outFile}"`);
}

Social Publisher services/worker/src/steps/socialPublisher.ts
ts
import fetch from 'node-fetch';

export async function publishToFacebook(pageId: string, token: string, message: string, imagePathOrUrl?: string, link?: string) {
  // if imagePathOrUrl is a local file, upload via multipart; here we assume URL
  const params = new URLSearchParams();
  if (imagePathOrUrl) params.append('url', imagePathOrUrl);
  params.append('caption', message + (link ? `\n${link}` : ''));
  params.append('access_token', token);
  const res = await fetch(`https://graph.facebook.com/${pageId}/photos`, {
    method: 'POST',
    body: params
  });
  const json = await res.json();
  if (json.error) throw new Error(JSON.stringify(json.error));
  return json;
}

export async function publishToInstagram(igUserId: string, token: string, caption: string, imageUrl: string) {
  // create media object
  const createParams = new URLSearchParams();
  createParams.append('image_url', imageUrl);
  createParams.append('caption', caption);
  createParams.append('access_token', token);
  const createRes = await fetch(`https://graph.facebook.com/${igUserId}/media`, {
    method: 'POST',
    body: createParams
  });
  const createJson = await createRes.json();
  if (createJson.error) throw new Error(JSON.stringify(createJson.error));
  // publish
  const publishParams = new URLSearchParams();
  publishParams.append('creation_id', createJson.id);
  publishParams.append('access_token', token);
  const publishRes = await fetch(`https://graph.facebook.com/${igUserId}/media_publish`, {
    method: 'POST',
    body: publishParams
  });
  return publishRes.json();
}

python
from flask import Flask, request, jsonify
import requests
import os
app = Flask(__name__)

# Configure backend for local LLM (text-generation-webui or other)
LLM_BACKEND = os.environ.get('LLM_BACKEND', 'http://localhost:7860')

@app.route('/generate', methods=['POST'])
def generate():
    data = request.json or {}
    prompt = data.get('prompt', '')
    # call text-generation-webui or other local endpoint
    try:
        resp = requests.post(f"{LLM_BACKEND}/api/generate", json={"prompt": prompt, "max_new_tokens": 200})
        resp.raise_for_status()
        out = resp.json()
        # assume out contains 'text'
        text = out.get('text') or out.get('generated_text') or str(out)
        return jsonify({"text": text})
    except Exception as e:
        return jsonify({"error": str(e)}), 500

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000)

4. سياسات النشر والفلترة وقواعد الموافقة
ملف التوثيق المقترح: dev_s_media/policies/publish_policy.md  
انسخ هذا المحتوى حرفياً

قواعد أساسية

لا تنشر أرقام هواتف أو بيانات شخصية. إن وُجدت استبدلها بـ "Contact via site".

كلمات ممنوعة: قائمة داخل الملف (أضف كلمات محلية إن لزم).

حالات تتطلب مراجعة بشرية: سعر مريب، صور تظهر لوحات أرقام، محتوى سياسي أو عنصري، أو أي ادعاء قانوني.

سياسة الموافقة

وضع افتراضي: AUTO_PUBLISH=false (شبه آلي).

تدفق الموافقة: توليد الاقتراح → Admin UI يعرض variant1/variant2 → مشرف يضغط Approve → Worker ينشر.

خيار تفعيل تلقائي: بعد 7 أيام من نجاح 100 منشور تجريبي بدون flags، يمكن تفعيل AUTO_PUBLISH=true.

سياسة التراجع

عند حذف الإعلان: ابحث عن social_posts وادعُ API لحذف المنشور إن أمكن. إن لم يكن متاحاً، انشر تحديث توضيحي على الصفحة.

6. سياسات الخطأ وإعادة المحاولة والمراقبة
ملف التوثيق المقترح: dev_s_media/checklist/deployment_checklist.md و monitoring/logging_config.json

Retry Policy

فشل مؤقت (5xx أو rate limit): إعادة محاولة 3 مرات مع exponential backoff (1s, 4s, 16s).

فشل دائم (4xx غير متعلق بالـ rate limit): علم failed وأرسل تنبيه للمشرف.

Rate Limits

احترس من حدود Meta: حدّد معدل نشر افتراضي 1 منشور/ثانية لكل app token، وراقب headers في الاستجابة.

مراقبة

سجّل كل طلب واستجابة في DB وملف لوج.

ربط Sentry لالتقاط الاستثناءات الحرجة.

Metrics to monitor: publish_success_rate, avg_time_to_publish, queue_depth, llm_latency.
