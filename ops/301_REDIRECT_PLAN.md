# 301 Redirect Implementation Plan\n\n## Overview\nWhen a listing's slug changes, we need to redirect from old slug → new canonical slug using HTTP 301 (permanent redirect).\n\n## Implementation Options\n\n### Option 1: Firebase Hosting Redirects (Recommended)\nFirebase Hosting supports redirects via `firebase.json`:\n\n```json\n{\n  \"hosting\": {\n    \"rewrites\": [\n      {\n        \"source\": \"/car/:sellerId/:listingId/:oldSlug\",\n        \"destination\": \"/redirect-slug/:sellerId/:listingId/:oldSlug\"\n      }\n    ]\n  }\n}\n```\n\nThen implement a Cloud Function:\n```typescript\nexport const redirectSlug = functions.https.onRequest(async (req, res) => {\n  const { sellerId, listingId, oldSlug } = req.params;\n  \n  const newSlug = await resolveOldSlug(oldSlug);\n  if (!newSlug) {\n    return res.status(404).send('Listing not found');\n  }\n  \n  res.redirect(301, `/car/${sellerId}/${listingId}/${newSlug}`);\n});\n```\n\n### Option 2: React Router Redirect\nIn React routing layer, detect slug mismatch:\n\n```typescript\nconst NumericCarDetailsPage = () => {\n  const { sellerId, listingId, slug } = useParams();\n  const [listing, setListing] = useState(null);\n\n  useEffect(() => {\n    loadListing(listingId).then(data => {\n      if (data.slug !== slug) {\n        // Redirect to canonical\n        window.location.href = `/car/${sellerId}/${listingId}/${data.slug}`;\n      }\n      setListing(data);\n    });\n  }, [listingId, slug]);\n\n  return listing ? <CarDetails listing={listing} /> : null;\n};\n```\n\n### Option 3: HTML Meta Tag Canonical\nFor SEO without hard redirect:\n\n```html\n<link rel=\"canonical\" href=\"https://koli.one/car/{sellerId}/{listingId}/{slug}\" />\n```\n\nGoogle crawlers will prefer canonical URL.\n\n## Recommendation\n**Use Option 1 + Option 3** for optimal SEO:\n- 301 redirects for users (tells browser & search engines \"permanently moved\")\n- Canonical tags for Googlebot (confirms primary URL)\n\n## Testing\n\n1. **Create listing with title \"BMW X5 2020\"**\n   - Slug generated: `bmw-x5-2020`\n   - URL: `https://koli.one/car/1/540/bmw-x5-2020`\n\n2. **Edit listing title to \"Beautiful BMW X5 2020 Sport\"**\n   - New slug: `beautiful-bmw-x5-2020-sport`\n   - Old slug history recorded\n   - URL should redirect:\n     ```\n     GET /car/1/540/bmw-x5-2020\n     Response: 301 Moved Permanently\n     Location: /car/1/540/beautiful-bmw-x5-2020-sport\n     ```\n\n3. **Verify with curl**\n   ```bash\n   curl -I -L https://koli.one/car/1/540/bmw-x5-2020\n   # Should follow redirect and land on new URL\n   ```\n\n4. **Check Firestore listing_slug_history**\n   ```\n   {\n     listingId: 'listing-abc',\n     oldSlug: 'bmw-x5-2020',\n     newSlug: 'beautiful-bmw-x5-2020-sport',\n     redirectTo: '/car/1/540/beautiful-bmw-x5-2020-sport',\n     changedAt: '2026-02-18T10:00:00Z'\n   }\n   ```\n\n## SEO Benefits\n\n- **Page Rank Flow**: 301 redirects pass ~99% of page rank (vs 302 = 0%)\n- **Backlink Updates**: Search engines update internal links\n- **URL Consolidation**: Prevents duplicate content penalties\n\n## Future: URL Shortener Integration\n\nOnce short links are live, 301 redirects also apply to short codes:\n```\nGET /s/abc123\nResolve to: /car/1/540/bmw-x5-2020-sport\nIf slug changes → short link points to new URL automatically\n```\n"
