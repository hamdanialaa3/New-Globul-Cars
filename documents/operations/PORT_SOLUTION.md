# 🔧 حل مشكلة البورت 3000 و Docker

## المشكلة
بعد استخدام Docker، البورت 3000 دائماً مشغول ولا تستطيع تشغيل المشروع.

## الحل السريع ✅

### الخيار 1: استخدام البورت الجديد الافتراضي (5173)
```bash
cd web
npm run dev
```
✅ سيعمل تلقائياً على البورت 5173

### الخيار 2: تحديد بورت مختلف يدويًا
```bash
cd web
npm run dev -- --port 3001
```

البورتات المتاحة:
- **5173** ⭐ افتراضي جديد (Vite)
- **3001** - بديل للـ 3000 القديم
- **3002** - بديل ثاني
- **8080** - بورت شهير
- **8000** - بديل آخر

### الخيار 3: استخدام السكريبت السهل
```powershell
.\start-dev.ps1
```
أو مع بورت معين:
```powershell
.\start-dev.ps1 -Port 3001
```

## تحرير البورت المشغول 🔓

إذا أردت فعلاً تحرير البورت 3000:

```powershell
# تشغيل السكريبت للتحقق والإيقاف
.\fix-port-3000.ps1
```

أو يدويًا:
```powershell
# البحث عن العملية
$port = 3000
$process = Get-NetTCPConnection -LocalPort $port -ErrorAction SilentlyContinue
if ($process) {
    Stop-Process -Id $process.OwningProcess -Force
}
```

## إيقاف حاويات Docker

إذا كان Docker يستخدم البورت:
```bash
# عرض جميع الحاويات
docker ps

# إيقافحاوية معينة
docker stop <container_name>

# إيقاف جميع الحاويات
docker stop $(docker ps -q)
```

## التغييرات المجراة ✨

1. ✅ **vite.config.ts**: تغيير البورت الافتراضي من 3000 إلى 5173
2. ✅ **vite.config.ts**: إضافة `strictPort: false` (يجرب بورت آخر إذا كان المطلوب مشغول)
3. ✅ **.env**: إضافة `PORT=5173` و `HOST=localhost`
4. ✅ **start-dev.ps1**: سكريبت سهل لاختيار البورت
5. ✅ **fix-port-3000.ps1**: سكريبت للتحقق من البورت وتحريره

## 💡 نصيحة مهمة
الآن البورت 5173 هو الافتراضي الجديد، وإذا كان مشغولاً Vite سيجرب بورت آخر تلقائياً!
