## Bulgarian Car Marketplace - Complete Profile System Plan

**التاريخ:** 18 أكتوبر 2025  
**الإصدار:** 2.0 (محدثة ومُفصّلة)  
**المشروع:** Globul Cars - Bulgarian Car Marketplace

## الدستور (Constitution)

### البيئة:
```
الموقع الجغرافي: جمهورية بلغاريا
اللغات: بلغاري + إنجليزي
العملة: يورو (EUR)
```

### القيود الفنية:
```
- الملفات: ≤300 سطر (تقسيم إذا زاد)
- لا تكرار (DRY principle)
- تحليل كل ملف قبل التعديل
- أيقونات من SVG فقط (https://www.svgrepo.com/)
- ممنوع: إيموجي نصية
- كل شيء real & production-ready
- تعليقات واضحة ومختصرة
- دوال معنّاة وواضحة
```

## الهدف الرئيسي

تحويل البروفايل إلى **مركز التحكم الشامل** لثلاثة أنواع من البائعين:
1. **Private Person** (شخصي)

## القسم 1: أنواع البروفايل والهوية البصرية مع LED Progress System

### مقدمة - نظام LED Progress Indicators:

جميع أنواع البروفايلات تحتوي على **شريط LED دائري** حول الصورة الشخصية يُظهر نسبة اكتمال البيانات (0% - 100%). هذا النظام:
- يحفّز المستخدم على إكمال بياناته
- يُظهر بصرياً ما هو ناقص
- يتغير لونه حسب نوع البروفايل
- يُكمل دورته عند 100%

**آلية العمل البسيطة:**
الشريط عبارة عن دائرة SVG تملأ تدريجياً بناءً على حساب نسبة الحقول المُكملة. كل حقل له وزن محدد (مثلاً: Email 20%, Phone 20%). عند إضافة/تحديث حقل، يُعاد حساب النسبة ويُحدّث الشريط فوراً بانيميشن سلس.

### 4.2 Trust Score Calculation (0-100):

```javascript
calculateTrustScore(seller) {
  let score = 0;
  
  // Base verifications
  if (seller.emailVerified) score += 10;
  if (seller.phoneVerified) score += 10;
  if (seller.idVerified) score += 20;
  if (seller.addressVerified) score += 10;
  
  // Business verifications (Dealer/Company)
  if (seller.businessVerified) score += 20;
  
  // Performance metrics
  const avgRating = seller.rating?.average || 0;
  score += (avgRating / 5) * 20; // max 20 points
  
  // Activity
  const yearsActive = getYearsActive(seller);
  score += Math.min(yearsActive * 2, 10); // max 10 points
  
  // Response metrics
  if (seller.stats?.responseRate > 0.8) score += 10;
  if (seller.stats?.avgResponseTime < 2 * 60 * 60 * 1000) score += 5; // <2h
  
  // Deal completion
  if (seller.stats?.completionRate > 0.7) score += 10;
  
  // Negative factors
  if (seller.disputes > 0) score -= seller.disputes * 5;
  if (seller.warnings > 0) score -= seller.warnings * 10;
  
  return Math.max(0, Math.min(100, score));
}
```

```typescript
// src/utils/listing-limits.ts

const PLAN_LIMITS = {
  free: 3,
  premium: 10,
  dealer_basic: 50,
  dealer_pro: 150,
  dealer_enterprise: -1,  // unlimited
  company_starter: 100,
  company_pro: -1,        // unlimited
  company_enterprise: -1,
  custom: -1
};

export async function canAddListing(userId: string): Promise<boolean> {
  const user = await getUser(userId);
  const limit = PLAN_LIMITS[user.plan.tier];
  
  if (limit === -1) return true;  // unlimited
  
  const activeCount = user.stats.activeListings || 0;
  return activeCount < limit;
}

// Usage in add-car flow:
if (!await canAddListing(userId)) {
  throw new Error('Listing limit reached. Upgrade your plan.');
}
```

```javascript
// firestore.rules - PRODUCTION-READY VERSION

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    
    // Helper Functions
    function isSignedIn() {
      return request.auth != null;
    }
    
    function isOwner(userId) {
      return isSignedIn() && request.auth.uid == userId;
    }
    
    function isAdmin() {
      return isSignedIn() && 
             request.auth.token.role == 'admin';
    }
    
    function isOwnerOrAdmin(userId) {
      return isOwner(userId) || isAdmin();
    }
    
    function hasRole(role) {
      return isSignedIn() && 
             request.auth.token.role == role;
    }
    
    function hasProfileType(type) {
      return isSignedIn() && 
             get(/databases/$(database)/documents/users/$(request.auth.uid)).data.profileType == type;
    }
    
    function canAddListing() {
      let user = get(/databases/$(database)/documents/users/$(request.auth.uid)).data;
      let limit = user.plan.tier == 'free' ? 3 :
                  user.plan.tier == 'premium' ? 10 :
                  user.plan.tier == 'dealer_basic' ? 50 :
                  user.plan.tier == 'dealer_pro' ? 150 : -1;
      return limit == -1 || user.stats.activeListings < limit;
    }
    
    // USERS COLLECTION
    match /users/{userId} {
      // Public profile info: anyone can read basic fields
      allow get: if isSignedIn();
      
      // Full profile: owner + admin only
      allow read: if isOwnerOrAdmin(userId);
      
      // Create: only on signup, limited fields
      allow create: if isOwner(userId) &&
                       request.resource.data.keys().hasAll(['email', 'createdAt']) &&
                       request.resource.data.createdAt == request.time;
      
      // Update: owner only, with restrictions
      allow update: if isOwner(userId) &&
                       // Cannot change these fields
                       request.resource.data.email == resource.data.email &&
                       request.resource.data.createdAt == resource.data.createdAt &&
                       // Verification status can only be changed by admin via Cloud Functions
                       (!request.resource.data.diff(resource.data).affectedKeys().hasAny(['verification']));
      
      // Delete: admin only (soft delete via Cloud Function preferred)
      allow delete: if isAdmin();
      
      // FCM Tokens subcollection
      match /fcmTokens/{tokenId} {
        allow read, write: if isOwner(userId);
      }
    }
    
    // LISTINGS COLLECTION
    match /listings/{listingId} {
      // Anyone can view published listings
      allow get: if resource.data.status == 'published';
      allow list: if request.query.limit <= 100;
      
      // Create: authenticated users who haven't exceeded their plan limit
      allow create: if isSignedIn() &&
                       canAddListing() &&
                       request.resource.data.ownerId == request.auth.uid &&
                       request.resource.data.createdAt == request.time;
      
      // Update: owner only, cannot change ownerId
      allow update: if isOwner(resource.data.ownerId) &&
                       request.resource.data.ownerId == resource.data.ownerId;
      
      // Delete: owner or admin
      allow delete: if isOwner(resource.data.ownerId) || isAdmin();
      
      // Analytics subcollection (aggregated daily to avoid hot writes)
      match /analytics/{date} {
        allow read: if isOwner(resource.data.ownerId) || isAdmin();
        allow write: if false; // Only Cloud Functions can write
      }
    }
    
    // REVIEWS COLLECTION
    match /reviews/{reviewId} {
      // Anyone can read published reviews
      allow read: if resource.data.status == 'published' || 
                     isOwnerOrAdmin(resource.data.targetUserId) ||
                     isOwner(resource.data.authorUserId) ||
                     isAdmin();
      
      // Create: authenticated users, one review per seller
      allow create: if isSignedIn() &&
                       request.resource.data.authorUserId == request.auth.uid &&
                       request.resource.data.createdAt == request.time &&
                       request.resource.data.status == 'pending' &&
                       // Verify stars is 1-5
                       request.resource.data.stars >= 1 &&
                       request.resource.data.stars <= 5;
      
      // Update/Delete: admin only (moderation)
      allow update, delete: if isAdmin();
    }
    
    // MESSAGES COLLECTION
    match /conversations/{conversationId} {
      // Read: participants only
      allow read: if isSignedIn() &&
                     request.auth.uid in resource.data.participantIds;
      
      // Create: authenticated users
      allow create: if isSignedIn() &&
                       request.auth.uid in request.resource.data.participantIds;
      
      // Update: participants only (for lastMessage, unreadCount)
      allow update: if isSignedIn() &&
                       request.auth.uid in resource.data.participantIds;
      
      // Messages subcollection
      match /messages/{messageId} {
        allow read: if isSignedIn() &&
                       request.auth.uid in get(/databases/$(database)/documents/conversations/$(conversationId)).data.participantIds;
        
        allow create: if isSignedIn() &&
                         request.resource.data.senderId == request.auth.uid &&
                         request.resource.data.timestamp == request.time;
        
        // Cannot update or delete messages (immutable)
        allow update, delete: if false;
      }
    }
    
    // FOLLOWS COLLECTION
    match /follows/{followId} {
      allow read: if isSignedIn();
      
      allow create: if isSignedIn() &&
                       request.resource.data.followerId == request.auth.uid &&
                       request.resource.data.createdAt == request.time;
      
      allow delete: if isSignedIn() &&
                       resource.data.followerId == request.auth.uid;
    }
    
    // VERIFICATION DOCUMENTS (admin-only access)
    match /verificationDocuments/{docId} {
      allow read: if isAdmin() || isOwner(resource.data.userId);
      allow create: if isSignedIn() && request.resource.data.userId == request.auth.uid;
      allow update, delete: if isAdmin();
    }
    
    // ADMIN-ONLY COLLECTIONS
    match /adminLogs/{logId} {
      allow read, write: if isAdmin();
    }
    
    match /systemConfig/{configId} {
      allow read: if isSignedIn();
      allow write: if isAdmin();
    }
  }
}
```

### 31.2 Incident Response Runbook

```
INCIDENT SEVERITY LEVELS:

P0 (Critical - Page immediately):
  - Platform completely down
  - Payment processing broken
  - Data breach suspected
  - Response: <15 minutes

P1 (High - Alert team):
  - Major feature broken (messaging, search)
  - Verification system down
  - Significant performance degradation
  - Response: <1 hour

P2 (Medium - Log and schedule):
  - Minor feature issues
  - UI bugs
  - Non-critical errors
  - Response: Next business day

COMMON INCIDENTS:

1. Payment Failures Spike
   Symptoms: Stripe webhook failures, dunning emails not sending
   Diagnosis:
     - Check Stripe dashboard for outages
     - Review Cloud Function logs for webhook processing errors
     - Verify webhook signature in environment variables
   Fix:
     - Re-send failed webhooks from Stripe dashboard
     - Manually update user plan.status if needed
     - Notify affected users
   Prevention:
     - Retry logic in webhook handler
     - Alert on >5% payment failure rate

2. Verification Backlog
   Symptoms: SLA breached, >100 pending verifications
   Diagnosis:
     - Check admin dashboard load times
     - Review EIK API availability
     - Check if admins are actively reviewing
   Fix:
     - Add temporary admin reviewers
     - Use EIK API batch endpoint if available
     - Communicate delays to users via email
   Prevention:
     - Auto-approve low-risk applications (with monitoring)
     - Scale admin team during peak times

3. Analytics Outage
   Symptoms: Dashboards not loading, aggregation lag >2 hours
   Diagnosis:
     - Check Cloud Function logs for aggregation jobs
     - Verify Firestore indexes exist
     - Check for hot documents (too many writes)
   Fix:
     - Manually trigger aggregation Cloud Function
     - Temporarily disable real-time updates
     - Backfill missing data from raw events
   Prevention:
     - Daily aggregation only (not real-time for analytics)
     - Monitor aggregation job completion
```

### 31.3 Backup and Restore

```
BACKUP STRATEGY:

Firestore:
  - Automatic: Firebase daily backups (built-in)
  - Manual: Weekly exports to Google Cloud Storage
  - Retention: 30 days (automatic), 1 year (manual exports)
  - Collections: users, listings, messages, reviews, verificationDocuments

Firebase Storage (images, documents):
  - Automatic: None (rely on user re-upload if needed)
  - Manual: Monthly sync to Google Cloud Storage bucket (different region)
  - Retention: 1 year
  - Critical documents: verificationDocuments (5 years)

Firebase Authentication:
  - Automatic: Firebase manages internally
  - Export: Monthly user list export (UID, email, metadata)
  - Retention: 1 year

Cloud Functions (code):
  - Version control: Git (primary backup)
  - Deployment history: Firebase retains last 10 deployments
  - Rollback: firebase deploy --only functions (previous version from Git)

RESTORE PROCEDURES:

Full Disaster Recovery:
  1. Provision new Firebase project
  2. Restore Firestore from latest backup
  3. Restore Storage from GCS sync
  4. Deploy Cloud Functions from Git
  5. Update DNS to point to new project
  6. Estimated time: 4-6 hours

Partial Data Loss:
  - Single collection: restore from daily backup (specific collection)
  - User data: contact user for verification documents re-upload
  - Analytics: backfill from raw events (if available)

TESTING:
  - Quarterly: Restore test (to separate Firebase project)
  - Verify: Data integrity, function deployments, security rules
  - Document: Any issues found, update runbook

RESPONSIBILITIES:
  - Dev team: Weekly manual exports
  - Admin: Monitor backup job success
  - Security team: Quarterly restore test
```
