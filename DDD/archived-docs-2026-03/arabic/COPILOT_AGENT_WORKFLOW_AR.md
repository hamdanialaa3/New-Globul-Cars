# 🤖 كيف تُعطي مهاماً لـ GitHub Copilot Agent؟

> **الجواب المختصر:** نعم — أنت تكتب المهمة، والـ Agent ينفّذها ويرفع النتيجة إلى GitHub تلقائياً.  
> **لكن:** الـ Agent لا يشتغل على حاسوبك المحلي — بل يشتغل على خوادم GitHub.

---

## 🗺️ كيف يعمل النظام بالكامل؟

```
أنت (على GitHub.com)
    │
    │  تكتب مهمة في Issue
    ▼
GitHub Copilot Agent
    │  يقرأ المهمة
    │  يحلّل ملفات المشروع
    │  يكتب الكود
    │  يختبره
    ▼
Pull Request جديد (تلقائياً)
    │
    ▼
أنت تراجع الـ PR
    │  تقبله → يُدمج في main
    │  أو ترفضه/تطلب تعديل
    ▼
تم! ✅
```

---

## 📝 الطريقة الصحيحة لإعطاء مهمة

### الخطوة 1 — أنشئ Issue جديدة

```
افتح: https://github.com/hamdanialaa3/New-Globul-Cars/issues/new
```

### الخطوة 2 — اكتب المهمة بوضوح

```
العنوان: [وصف مختصر للمهمة]

المحتوى:
- ماذا تريد بالضبط؟
- أين في الكود؟ (اذكر اسم الملف أو الصفحة إن عرفته)
- ما هو السلوك المتوقع؟
- هل هناك قواعد أو قيود خاصة؟
```

**مثال مهمة جيدة:**
```
العنوان: إضافة زر "مشاركة" في صفحة تفاصيل السيارة

المحتوى:
أريد إضافة زر "مشاركة" في صفحة عرض تفاصيل السيارة بحيث:
- يظهر بجانب زر "المفضلة"
- عند الضغط عليه يُنسخ رابط الصفحة في الـ clipboard
- تظهر رسالة تأكيد "تم نسخ الرابط" لمدة 2 ثانية
- الرسالة باللغتين (عربي/إنجليزي)
الملف المتوقع: src/pages/car-details/
```

### الخطوة 3 — عيّن المهمة للـ Copilot

بعد إنشاء الـ Issue، في نفس الصفحة:

```
في القائمة اليمنى → Assignees → ابحث عن "Copilot" → اختره
```

أو في تعليق على الـ Issue اكتب:
```
@github-copilot work on this
```

### الخطوة 4 — انتظر الـ Pull Request

```
بعد دقائق (أو ساعة للمهام الكبيرة)، ستصلك رسالة على GitHub:
"Copilot opened a pull request for this issue"
```

اذهب إلى: `https://github.com/hamdanialaa3/New-Globul-Cars/pulls`

### الخطوة 5 — راجع واقبل (أو اطلب تعديل)

```
1. افتح الـ PR
2. اضغط "Files changed" لرؤية ما تغيّر
3. إذا كان صحيحاً: اضغط "Merge pull request"
4. إذا تريد تعديل: اكتب تعليقاً يشرح ما يجب تغييره
   → سيُحدّث Copilot الـ PR تلقائياً
```

## ✅ ما يستطيع الـ Agent فعله

| المهمة | هل يستطيع؟ |
|--------|------------|
| إضافة ميزة جديدة | ✅ نعم |
| إصلاح bug محدد | ✅ نعم |
| تحديث نص أو ترجمة | ✅ نعم |
| إضافة صفحة جديدة | ✅ نعم |
| كتابة اختبارات | ✅ نعم |
| تحسين الأداء | ✅ نعم |
| تحديث التوثيق | ✅ نعم |
| الوصول إلى Firebase/API مباشرة | ❌ لا |
| النشر (deploy) على الخادم | ❌ لا (يحتاج إعداد CI/CD) |

## ⚠️ ملاحظات مهمة

### 1. الـ Agent لا يشتغل على حاسوبك
الـ Agent يعمل على خوادم GitHub — لا يحتاج وصولاً لحاسوبك ولا يمكنه ذلك.  
**هذا أمان، ليس قصوراً.**

### 2. أنت المراجع الأخير دائماً
لا يُدمج أي كود في المشروع بدون موافقتك على الـ PR.  
لا تقبل الـ PR دون مراجعة الكود.

### 3. المهمة الواضحة = نتيجة أفضل
كلما كانت المهمة أكثر تفصيلاً، كانت نتيجة الـ Agent أدق.  
المهمة الغامضة تُنتج كوداً قد لا يطابق ما تريد.

### 4. يمكنك طلب تعديل بعد رؤية الـ PR
إذا لم يكن الكود كما تريد، اكتب في تعليق على الـ PR:
```
هذا ليس ما قصدته. أريد X بدلاً من Y لأن Z.
```
سيُعيد الـ Agent المحاولة بناءً على تعليقك.

## 🔄 مقارنة: Agent على GitHub مقابل Copilot في VS Code

| | GitHub Copilot Agent | GitHub Copilot في VS Code |
|---|---|---|
| **أين يعمل؟** | على خوادم GitHub | على حاسوبك |
| **كيف تعطيه مهمة؟** | GitHub Issue | كتابة في Chat |
| **من ينفّذ الكود؟** | هو تلقائياً | يقترح، أنت تنفّذ |
| **النتيجة** | PR جاهز للمراجعة | مساعدة في الكتابة |
| **متى تستخدمه؟** | مهام كاملة محددة | برمجة يومية وأسئلة |
